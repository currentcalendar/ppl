## 1. Antes de empezar

### Levantar el entorno

```
docker compose -f docker-compose.yaml up -d --build
```

### Obtener un token válido para pruebas

Usaremos un usuario del seeder (`ana@example.com` / `password123`).

```powershell
$loginResponse = Invoke-WebRequest -Uri "http://localhost:8000/api/v1/token/" -Method POST -ContentType "application/json" -Body '{"username":"ana_garcia","password":"password123"}'
$token = ($loginResponse.Content | ConvertFrom-Json).access
Write-Host "Token obtenido: $token"
```

### Función auxiliar reutilizable

Cópiala en tu terminal:

```powershell
function Invoke-Test {
    param([int]$i, [string]$Uri, [string]$Method = "GET", [string]$Body = $null, [string]$Token = $null)
    try {
        $headers = @{}
        if ($Token) { $headers["Authorization"] = "Bearer $Token" }
        $params = @{ Uri = $Uri; Method = $Method; Headers = $headers }
        if ($Body) { $params["ContentType"] = "application/json"; $params["Body"] = $Body }
        $response = Invoke-WebRequest @params
        Write-Host "Request $i`: $($response.StatusCode)"
    } catch {
        Write-Host "Request $i`: $($_.Exception.Message)"
    }
}
```

## 2. Pruebas de Rate Limiting por Scope

Ejecuta los bucles para comprobar los límites de cada scope. Deberías ver un error `429 Too Many Requests` una vez superado el límite.

### 2.1 Auth (5 peticiones/minuto)

Límite aplicado a todos los endpoints de autenticación (Login, Registro, Recuperación), de forma compartida.

```powershell
for ($i = 1; $i -le 7; $i++) { Invoke-Test -i $i -Uri "http://localhost:8000/api/v1/token/" -Method POST -Body '{"email":"ana@example.com","password":"wrong"}' }
```

### 2.2 Anon (20 peticiones/minuto)

Límite general para usuarios no autenticados en endpoints públicos.

```powershell
for ($i = 1; $i -le 22; $i++) { Invoke-Test -i $i -Uri "http://localhost:8000/api/v1/calendars/list/" }
```

_Si tienes un token, un usuario autenticado usa el scope User, no Anon._

### 2.3 User (100 peticiones/minuto)

Límite general para peticiones autenticadas.

```powershell
for ($i = 1; $i -le 102; $i++) { Invoke-Test -i $i -Uri "http://localhost:8000/api/v1/calendars/list/" -Token $token }
```

### 2.4 Heavy (15 peticiones/minuto)

Límite estricto para endpoints pesados de base de datos (e.g. Radar).

```powershell
for ($i = 1; $i -le 17; $i++) { Invoke-Test -i $i -Uri "http://localhost:8000/api/v1/radar/?lat=40.4168&lon=-3.7038" -Token $token }
```

## 5. Endpoints pesados — scope `heavy` (15/min)

### 5.1 Radar
**Resultado esperado:** Requests 1-15 pasan, 16 y 17 devuelven `429`.

```powershell
for ($i = 1; $i -le 17; $i++) {
    Invoke-Test -i $i `
        -Uri "http://localhost:8000/api/v1/radar/" `
        -Token $token
}
```

### 5.2 Recommended calendars

```powershell
for ($i = 1; $i -le 17; $i++) {
    Invoke-Test -i $i `
        -Uri "http://localhost:8000/api/v1/recommendations/calendars/" `
        -Token $token
}
```

### 5.3 Recommended events

```powershell
for ($i = 1; $i -le 17; $i++) {
    Invoke-Test -i $i `
        -Uri "http://localhost:8000/api/v1/recommendations/events/" `
        -Token $token
}
```

### 5.4 Verificar que el contador `heavy` es independiente del `user`

Un usuario puede agotar el límite heavy (15/min) sin que afecte a su límite user (100/min).

```powershell
# Agota el límite heavy en radar
for ($i = 1; $i -le 16; $i++) {
    Invoke-Test -i $i `
        -Uri "http://localhost:8000/api/v1/radar/" `
        -Token $token
}

# Un endpoint normal (scope User) no debe verse afectado por el límite Heavy
Invoke-Test -i 18 -Uri "http://localhost:8000/api/v1/calendars/list/" -Token $token
```

## 3. Resumen y Depuración

### 3.1 Valores Esperados

| Scope   | Endpoint ejemplo        | Límite  | 429 en   | Clave en Redis             |
| ------- | ----------------------- | ------- | -------- | -------------------------- |
| `auth`  | `/api/v1/token/`        | 5/min   | 6ª pet.  | `throttle_auth_<IP>`       |
| `anon`  | Pública / sin token     | 20/min  | 21ª pet. | `throttle_anon_<IP>`       |
| `user`  | Privada / con token     | 100/min | 101ª pet | `throttle_user_<user_id>`  |
| `heavy` | `/api/v1/events/radar/` | 15/min  | 16ª pet. | `throttle_heavy_<user_id>` |

### 3.2 Comandos Redis Útiles

```powershell
# Ver keys de throttle activas
docker exec -it redis_container redis-cli -n 1 KEYS "*throttle*"

# Ver TTL de una key (segundos para reset)
docker exec -it redis_container redis-cli -n 1 TTL ":1:throttle_auth_172.22.0.1"

# Limpiar cache/contadores (solo en desarrollo)
docker exec -it redis_container redis-cli -n 1 FLUSHDB
```

> **Nota:** Si PowerShell devuelve `Unknown Status`, o un error al hacer `Invoke-WebRequest` con 429, fíjate en el mensaje de error: indica que has recibido un 429 correctamente.
