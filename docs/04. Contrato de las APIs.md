# Contrato de las APIs

## Propósito

El presente documento tiene la intención de informar a los desarrolladores de Current sobre la arquitectura de API REST del proyecto. A continuación se encuentran las rutas a emplear separadas por módulo.

> **Nota:** Todas las rutas comienzan desde la raíz del servidor (p. ej. `http://localhost:8000`). Los errores estándar devuelven `{ "errors": ["..."] }` o `{ "error": "..." }` con el código HTTP correspondiente.

| :--- | :--- | :--- | :--- |
| `POST` | `/api/v1/auth/register/` | Registro de nuevo user | `201` `{ "message": "User registrado exitosamente", "user": { id, username, email, ... } }` |
| `POST` | `/api/v1/auth/login/` | Inicio de sesión | _Pendiente de implementar_ |
| `POST` | `/api/v1/auth/logout/` | Cierre de sesión | _Pendiente de implementar_ |
| `POST` | `/api/v1/auth/password_reset/` | Solicitar email de restablecimiento | _Pendiente de implementar_ |
| `POST` | `/api/v1/auth/password_reset/confirm/` | Cambiar contraseña con el token del email | _Pendiente de implementar_ |
| `PUT` | `/api/v1/users/me/` | Editar perfil propio | `200` Datos del user actualizados (UserSerializer) |
| `DELETE` | `/api/v1/users/me/` | Eliminar cuenta propia | `202` `{ "message": "User eliminado satisfactoriamente" }` |
| `GET` | `/api/v1/users?search=<query>` | Buscar users por username, email o pronouns | `200` `[ { id, username, email, pronouns, bio, photo, total_seguidores, total_following, total_calendarios_following }, ... ]` |
| `POST` | `/api/v1/users/{id}/follow/` | Seguir / dejar de seguir a un user | `200` `{ "user": <id>, "followed": <bool> }` |
| `POST` | `/api/v1/users/{id}/block/` | Bloquear a un user | _Pendiente de implementar_ |
| Método | Ruta | Auth | Descripción | Respuesta (éxito) |
| :--- | :--- | :--- | :--- | :--- |
| `POST` | `/api/v1/auth/register/` | No | Registro de nuevo user | `200` `{ "message": "User registered succesfully", "user": { photo, email, username, pronouns, link, bio, total_followers, total_following, subscribed_calendars } }` |
| `POST` | `/api/v1/token/` | No | Obtener JWT (access + refresh) | `200` `{ "access": "<token>", "refresh": "<token>" }` |
| `POST` | `/api/v1/token/refresh/` | No | Renovar access token | `200` `{ "access": "<token>" }` |
| `POST` | `/api/v1/auth/recover-password/` | No | Solicitar email de restablecimiento | _Pendiente de implementar_ |
| `POST` | `/api/v1/auth/set-new-password/` | No | Cambiar contraseña con token del email | _Pendiente de implementar_ |
| `POST` | `/api/v1/auth/validate-reset-token/` | No | Validar token de reset | _Pendiente de implementar_ |
| `GET` | `/api/v1/users/me/` | Sí | Obtener perfil propio | `200` `{ id, username, email, pronouns, bio, link, photo, total_followers, total_following, calendars, following_calendars }` |
| `PATCH` | `/api/v1/users/me/edit/` | Sí | Editar perfil propio | `200` `{ "message": "Profile updated correctly", "user": { username, photo, pronouns, link, bio } }` |
| `DELETE` | `/api/v1/users/me/delete/` | Sí | Eliminar cuenta propia | `202` `{ "message": "User deleted successfully" }` |
| `GET` | `/api/v1/users/search/?search=<query>` | No | Buscar users por username, email o pronouns | `200` `[ { id, username, pronouns, bio, photo, link, total_followers, total_following, is_following }, ... ]` |
| `GET` | `/api/v1/users/<pk>/` | No | Perfil público de un user por ID | `200` `{ id, username, pronouns, bio, photo, link, total_followers, total_following, is_following, public_calendars }` |
| `GET` | `/api/v1/users/by-username/<username>/` | No | Perfil público de un user por username | `200` igual que el anterior |
| `POST` | `/api/v1/users/<pk>/follow/` | Sí | Seguir / dejar de seguir (toggle) | `200` `{ "user_id": <int>, "followed": <bool> }` |
| `GET` | `/api/v1/users/<pk>/followers/` | No | Listar seguidores de un user | `200` `[ { id, username, pronouns, bio, photo, link, total_followers, total_following, is_following }, ... ]` |
| `GET` | `/api/v1/users/<pk>/following/` | No | Listar a quién sigue un user | `200` `[ { id, username, pronouns, bio, photo, link, total_followers, total_following, is_following }, ... ]` |
| `GET` | `/api/v1/users/<pk>/followed_calendars/` | No | Calendarios públicos a los que está suscrito un user | `200` `[ { id, name, description, cover, created_at }, ... ]` |
| `POST` | `/api/v1/users/{id}/block/` | Sí | Bloquear a un user | _Pendiente de implementar_ |

> **Nota sobre `is_following`:** Este campo indica si el usuario autenticado sigue al user listado. Devuelve `false` en peticiones anónimas (sin token).

**Body de `POST /api/v1/auth/register/`:**
```json
{
  "username": "string",
  "email": "string",
  "password": "string",
  "password2": "string"
}
```

**Body de `POST /api/v1/token/`:**
```json
{
  "username": "string",
  "password": "string"
}
```

**Body de `PATCH /api/v1/users/me/edit/`** (todos opcionales):
```json
{
  "username": "string",
  "photo": "<archivo>",
  "pronouns": "string",
  "link": "string",
  "bio": "string"
}
```

**Errores comunes en endpoints de follow:**
- `400` `{ "error": "You cannot follow yourself" }` — al intentar seguirse a uno mismo
- `404` `{ "error": "User not found" }` — si el `pk` no existe
- `401` `{ "detail": "Authentication credentials were not provided." }` — sin token en endpoints protegidos

## Gestión de calendars

| Método | Ruta | Descripción | Respuesta (éxito) |
| :--- | :--- | :--- | :--- |
| `POST` | `/api/v1/calendars/create/` | Crear un nuevo calendar | `201` `{ id, origin, external_id, name, description, privacy, creator_id, created_at, likes_count, liked_by_me, cover }` |
| `GET` | `/api/v1/calendars/list/` | Listar calendars (con filtros opcionales `?q=<name>&privacy=<PRIVATE\|PUBLIC>`) | `200` `[ { id, name, description, privacy, origin, creator_id, creator_username, created_at, likes_count, liked_by_me, cover }, ... ]` |
| `GET` | `/api/v1/calendars/my-calendars/` | Listar mis calendars | `200` `[ { ...calendar_summary } ]` |
| `GET` | `/api/v1/calendars/subscribed/` | Listar calendars a los que estoy suscrito | `200` `[ { ...calendar_summary } ]` |
| `PUT` | `/api/v1/calendars/{id}/publish/` | Publicar un calendar (cambia privacy a PUBLIC) | `200` `{ id, name, description, privacy, origin, creator, created_at }` |
| `PUT` / `PATCH` | `/api/v1/calendars/{id}/edit/` | Editar name, descripción o privacy de un calendar | `200` `{ id, name, description, privacy }` |
| `DELETE` | `/api/v1/calendars/{id}/delete/` | Eliminar un calendar (solo el creator) | `204` Sin contenido |
| `POST` | `/api/v1/calendars/{id}/like/` | Dar/Quitar like (toggle) a un calendar | `200` `{ calendar_id, liked, likes_count, liked_by_me }` |

**Body de `POST /api/v1/calendars/create/`:**
```json
{
  "name": "string",
  "description": "string (opcional)",
  "privacy": "PRIVATE | PUBLIC (opcional, default: PRIVATE)",
  "origin": "CURRENT | GOOGLE | APPLE (opcional, default: CURRENT)",
  "external_id": "string (opcional)"
}
```

`calendar_summary`:
```json
{
  "id": 1,
  "name": "string",
  "description": "string",
  "privacy": "PRIVATE|PUBLIC",
  "origin": "CURRENT|GOOGLE|APPLE",
  "creator_id": 2,
  "creator_username": "string",
  "created_at": "datetime",
  "likes_count": 0,
  "liked_by_me": false,
  "cover": "url|null"
}
```

**Body de `PUT/PATCH /api/v1/calendars/{id}/edit/`:**
```json
{
  "name": "string (opcional)",
  "description": "string (opcional)",
  "privacy": "PRIVATE | PUBLIC (opcional)"
}
```

| Método | Ruta | Descripción | Respuesta (éxito) |
| :--- | :--- | :--- | :--- |
| `POST` | `/api/v1/events/create/` | Crear un nuevo event | `201` `{ id, title, description, place_name, date, time, recurrence, external_id, calendars: [int], created_at }` |
| `PUT` | `/api/v1/events/{id}/edit/` | Editar un event existente | `200` `{ id, title, description, place_name, date, time, recurrence, external_id, calendars: [int], created_at }` |
| `GET` / `DELETE` | `/api/v1/events/{id}/delete/` | Detalle / borrado de event | _Pendiente de implementar_ |
| `POST` | `/api/v1/events/asign-to-calendar/` | Asignar un event a un calendar | `200` `{ "mensaje": "Event '<title>' asignado al calendar '<name>'" }` |
| `DELETE` | `/api/v1/events/deasign-from-calendar/` | Desasignar un event de un calendar | `200` `{ "mensaje": "Event '<title>' desasignado del calendar '<name>'" }` |

**Body de `POST /api/v1/events/create/`:**
```json
{
  "title": "string",
  "date": "YYYY-MM-DD",
  "time": "HH:MM:SS",
  "calendars": [<calendario_id>, ...],
  "creator_id": "<int>",
  "description": "string (opcional)",
  "place_name": "string (opcional)",
  "latitud": "<float> (opcional)",
  "longitud": "<float> (opcional)",
  "recurrence": "string (opcional)",
  "external_id": "string (opcional)"
}
```

**Body de `POST /api/v1/events/asign-to-calendar/` y `DELETE /api/v1/events/deasign-from-calendar/`:**
```json
{
  "evento_id": "<int>",
  "calendario_id": "<int>"
}
```

## Importación / Exportación de calendars

| Método | Ruta | Descripción | Respuesta (éxito) |
| :--- | :--- | :--- | :--- |
| `GET` | `/api/v1/google-auth` | Iniciar flujo OAuth2 de Google Calendar | `302` Redirección a Google OAuth |
| `GET` / `POST` | `/api/v1/calendars/import-google-calendar/` | Importar events desde Google Calendar (requiere sesión OAuth2 activa) | `200` `{ "message": "Eventos importados exitosamente", "count": <int> }` |
| `POST` | `/api/v1/calendars/import-ios-calendar/` | Importar calendar desde iOS Calendar vía URL webcal | `200` `{ "message": "Calendar iOS importado exitosamente", "count": <int> }` |
| `POST` | `/api/v1/calendars/import-ics/` | Importar calendar desde archivo `.ics` subido | `200` `{ "message": "Archivo ICS importado exitosamente" }` |
| `GET` | `/api/v1/calendars/{id}/export/` | Exportar un calendar a formato `.ics` | `200` Archivo adjunto `text/calendar` |

**Body de `POST /api/v1/calendars/import-ios-calendar/`:**
```json
{
  "webcal_url": "webcal://... o https://...",
  "user": "<int>",
  "privacy": "PRIVATE | PUBLIC (opcional, default: PRIVATE)"
}
```

**Body de `POST /api/v1/calendars/import-ics/` (multipart/form-data):**
```
file: <archivo .ics>
user: <int>
privacy: PRIVATE | PUBLIC (opcional, default: PRIVATE)
```
