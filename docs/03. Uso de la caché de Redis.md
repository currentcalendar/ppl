# Uso de la caché de Redis

## ¿Qué es Redis aquí?

Redis actúa como un **caché** (almacenamiento temporal en memoria) para evitar consultas repetidas a la base de datos.

## Cómo funciona:

### 1. **Verificar si existe en caché**

```python
from django.core.cache import cache

cache_key = "sevilla_point_data"
cached_data = cache.get(cache_key)
if cached_data:
    return Response({
        "source": "Redis (Cache)",
        "data": cached_data
    })
```

- Primero busca los datos en Redis con una clave (`"sevilla_point_data"`)
- Si los encuentra, devuelve inmediatamente sin tocar la BD

### 2. **Si no está en caché, consultar BD**

- Si no hay datos en caché, consulta PostgreSQL normalmente
- Si el objeto ya existe, lo recupera; si no, lo crea

### 3. **Guardar en caché**

```python
result = {
    "id": obj.id,
    "name": obj.name,
    "coordenadas": {
        "longitude": obj.punto_geografico.x,
        "latitude": obj.punto_geografico.y
    },
    "created_in_db": created,
    "timestamp": str(obj.created_at)
}

cache.set(cache_key, result, 60)
```

- Almacena el resultado en Redis con una **duración de 60 segundos**
- Las siguientes solicitudes en esos 60 segundos usarán Redis (mucho más rápido)

## Flujo resultante:

```
1ª consulta  → BD → Redis (60s) → Respuesta
2ª consulta  → Redis ✓ (instantáneo)
3ª consulta  → Redis ✓ (instantáneo)
Pasados 60s  → BD → Redis (otros 60s) → Respuesta
```

## Beneficio:

Reduce carga en la Base de Datos y mejora significativamente la velocidad de respuesta.
